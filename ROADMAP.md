## 🔄 **버전 관리 전략**

### 📦 **버전 명명 규칙 (Semantic Versioning)**
- **MAJOR.MINOR.PATCH** (예: 1.2.3)
  - **MAJOR**: 주요 기능 추가 또는 구조적 변경
  - **MINOR**: 새로운 기능 추가 (하위 호환성 유지)
  - **PATCH**: 버그 수정 및 소규모 개선

### 🚀 **릴리즈 사이클**
- **주요 업데이트**: 3-4개월 주기
- **기능 업데이트**: 1-2개월 주기  
- **버그 수정**: 필요 시 즉시

### 📋 **업데이트 배포 프로세스**
1. **개발 완료** → 내부 테스트
2. **베타 버전** → 팀 내 테스트 (1주일)
3. **릴리즈 후보** → 최종 검토
4. **정식 배포** → Google Drive 자동 업데이트

## 🛠️ **개발 명령어**

### 📈 **버전 업데이트**
```bash
# 기능 추가 (1.0.0 → 1.1.0)
npm run version:minor

# 주요 업데이트 (1.0.0 → 2.0.0)  
npm run version:major

# 버그 수정 (1.0.0 → 1.0.1)
npm run version:patch

# 전체 릴리즈 (빌드 + 배포 + 업로드)
npm run release
```

### 🔧 **개발 워크플로우**
```bash
# 개발 환경 실행
npm run dev          # Next.js 서버
npm run electron     # Electron 앱

# 빌드 및 배포
npm run build        # Next.js 빌드
npm run dist         # EXE 파일 생성
npm run upload-version  # Google Drive 업로드
```
